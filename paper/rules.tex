We adapt an advice mechanism introduced for \DFA in~\cite{ecai25} to \DVPA. 
The main aim of advice is to 
provide the learning algorithm with prior knowledge about the learned language, which is used to reduce the number of equivalence queries.

First, we present the general framework, which carries over from the regular case seamlessly.
Next, We discuss which rewriting rules are compatible with \VPLs and how to extend the SRS formalism to capture natural 
properties of \VPLs.

\subsection{Learning with advice}
We adapt the framework of active learning with advice for \DFA~\cite{ecai25} to \DVPA. 
The advice supplied to the learning  algorithm is expressed via string rewriting systems (SRSs), 
which are connected to visibly pushdown languages through the following notions of consistency: 


\begin{definition}[\cite{ecai25}]
\label{def:consistent}
Let $\vAlph$ be a pushdown alphabet, $\rew$ be an SRS over $\vAlph$ and $\lang$ be a visibly pushdown language over $\vAlph$. 
We say that the SRS $\rew$ is
\begin{itemize}
\item {(fully) consistent} with $\lang$ if and only if
for all words $x,y$, if $x \rewrites_{\rew} y$, then $x \in \lang \Leftrightarrow y \in \lang$.  
\item {positively consistent} with $\lang$ if and only if for all  $x,y$, if $x \rewrites_{\rew} y$,
then $x \in \lang \Rightarrow y \in \lang$.
\item {negatively consistent} with $\lang$ if and only if for all  $x,y$, if $x \rewrites_{\rew} y$, then $x \notin \lang \implies y \notin \lang$.  
\end{itemize}
\end{definition}

Note that an SRS $\rew$ is fully consistent with $\lang$ if and only if it is both positively and negatively consistent.
%While more granular than standard consistency, positive and negative consistency are harder to infer.

With these notions of consistency in place, we can now present the active learning with advice framework.

\Paragraph{Framework}
The active \DVPA learning with advice problem is defined as follows.
The algorithm is given an \emph{oracle} $\oracle$ answering membership and equivalence queries about the target visibly pushdown language $\lang$
 (and possibly other queries), and three \emph{advice} SRSs:
 \begin{itemize}
 \item  $\rew_{=}$, which is consistent with $\lang$, and 
 \item $\rew_{+}$, $\rew_{-}$, which are respectively positively and negatively consistent with $\lang$.
\end{itemize}  
The algorithm's the task is to construct a \DVPA recognizing $\lang$. 
The advice is used to approximate equivalence checks, thereby reducing the number of equivalence queries posed to the oracle.
Therefore, it can be integrated with any framework for active learning of visibly pushdown languages~\cite{DBLP:conf/icgi/BarbotBFHKLN0Y21,DBLP:journals/pacmpl/JiaT24} or \DVPA~\cite{mfcs22} with possibly different types of queries 
provided that the oracle answers at least  membership and equivalence queries.
In particular, the oracle may also answer additional types of queries as in~\cite{mfcs22}.


\Paragraph{Using advice}
%We can reduce the number of equivalence queries without affecting the learning algorithm.
%When the learning algorithm attempts to ask the oracle an equivalence query,
We intercept  equivalence queries posed by the learning algorithm and first check whether the advice SRS $\rew$ is consistent (resp., positively consistent or negatively consistent) with
the language of a given candidate automaton $\aut$. 
We forward $\aut$ to the oracle, only if it is consistent 
Otherwise, if it is inconsistent, without the oracle we conclude that $\lang(\aut)$ differs from the target language with which $\rew$ is consistent (resp., positively/negatively consistent).
It suffices to compute a pair of words violating consistency (resp., positive/negative consistency), i.e., 
words $x,y$ such that $x \rewrites_{\rew} y$ but exactly of $x,y$ belongs to $\lang$.
Then, one of the words $x, y$ must be a counterexample to $\lang(\aut)$ being equal to the target language. 
Which one it is can be determined with a single membership query.


\subsection{Rewrite rules compatible with VPLs}

While the notion of consistency with the language is the same for \DVPA as for \DFA, the key difference lies in the choice of rewrite rules. 
With regular languages, any pair of words is a valid string rewrite rule, but words in visibly pushdown languages 
have an implicit structure induced by the matching relation between calls and returns.
The intuition behind rewrite rules is that words are rewritten to other words that are equivalent with respect to a given language. 
Rewrite rules should therefore preserve the call--return matching outside the rewritten subword. 
In other words, rewriting should act locally and not affect the rest of the word.
While this is trivially true for regular words, it is not for pushdown words. 
For instance, by rewriting 
\(
\textcolor{Green}{c_1} \textcolor{red}{c_2} \textcolor{blue}{c_3} \textcolor{blue}{r_3} \textcolor{red}{r_2} \textcolor{Green}{r_1}
\) with the rule $c_3 r_3 \to r_3 c_3$
we get a well-matched word with a different matching relation (color coded) 
\(
\textcolor{Green}{c_1} \textcolor{red}{c_2} \textcolor{red}{r_3} \textcolor{blue}{c_3} \textcolor{blue}{r_2}  \textcolor{Green}{r_1}.
\)
%the matching $(c_1, r_1), (c_2, r_2), (c_3, r_3)$ changes to $(c_1, r_1), (c_2, r_3), (c_3, r_2)$


\begin{definition}
A rewrite rule $\lhs \to \rhs$ is {matching-preserving} over $\vAlph$ if and only if for all words $w,w' \in \vAlph^*$
such that $w = x \lhs y$ and $w' = x \rhs y$, the matching relation between calls from $x$ and returns from $y$ in $w$
coincides with the matching relation between calls from $x$ and returns from $y$ in $w'$.
An SRS $\rew$ is {matching-preserving} if all rules in $\rew$ are matching-preserving. 
\end{definition}

Matching-preserving rules do not break the matching between calls and returns that are not in the rewritten fragment of the word.
A prominent subclass of matching-preserving rules is the set of \emph{well-matched rules}, 
which arise naturally and enjoy more efficient algorithms for checking consistency:

\begin{definition}
A rewrite rule $\lhs \to \rhs$ is well-matched over $\vAlph$ if and only if words $\lhs$ and $\rhs$ are well-matched.
An SRS $\rew$ is {well-matched} if all rules in $\rew$ are well-matched.
\end{definition}

Every well-matched rule is matching-preserving. However, not every matching-preserving rule is well-matched: 
for instance, a rewrite rule $c_1 c_2 r_2 \to c_1$
is matching-preserving, but it is not well-matched. Most of our examples are well-matched rules.

\subsection{Rewrite rules with variables}
While we have restricted admissible rules to those that affect pushdown words locally, we face another problem. 
Natural properties of pushdown languages cannot be expressed with string rewriting rules, 
which we illustrate with the example below.

\begin{example}
\newcommand{\OpenDelOne}{\ensuremath{\boldsymbol{\langle}}\,}
\newcommand{\CloseDelOne}{\ensuremath{\boldsymbol{\rangle}}\,}
\newcommand{\OpenDelTwo}{\ensuremath{\boldsymbol{[}}\,}
\newcommand{\CloseDelTwo}{\ensuremath{\boldsymbol{]}}\,}
Consider the Dyck language $\lang_2$ of well-matched brackets over two pairs of brackets: \( \set{ \OpenDelOne, \CloseDelOne} \) and \( \set{\OpenDelTwo, \CloseDelTwo} \). 
The language $\lang_2$ satisfies the property that the order of \OpenDelOne and \OpenDelTwo is irrelevant provided that 
each opening bracket is matched by its corresponding closing bracket. 
For instance, in every word substituting $\OpenDelOne \OpenDelTwo \CloseDelTwo \CloseDelOne$ with 
 $\OpenDelTwo  \OpenDelOne \CloseDelOne \CloseDelTwo$ neither breaks the matching between other brackets nor fixes unmatched brackets.
Consequently,
  $\OpenDelOne \OpenDelTwo \CloseDelTwo \CloseDelOne \to \OpenDelTwo  \OpenDelOne \CloseDelOne \CloseDelTwo$ rule is consistent with
  $\lang_2$. 
Unfortunately, this property cannot be expressed by any finite set of string rewriting rules as it requires  exchanging arbitrarily distant letters; for example, for any $n \geq 1$, the word 
\(  \OpenDelOne \underbrace{\OpenDelTwo \ldots \OpenDelTwo}_\text{n} \OpenDelOne \CloseDelOne \underbrace{\CloseDelTwo \ldots \CloseDelTwo}_\text{n} \CloseDelOne \) 
is equivalent to 
\(  \OpenDelOne \OpenDelOne \underbrace{\OpenDelTwo \ldots \OpenDelTwo}_\text{n} \underbrace{\CloseDelTwo \ldots \CloseDelTwo}_\text{n} \CloseDelOne \CloseDelOne. \) 
This property can, however, be expressed using a word variable $X$ ranging over well-matched words.
Then, the rule
\begin{align*}
&& \OpenDelOne \OpenDelTwo X \CloseDelTwo \CloseDelOne \to  \OpenDelTwo  \OpenDelOne X \CloseDelOne \CloseDelTwo
\end{align*}
expresses that the relative order of \OpenDelOne and \OpenDelTwo is irrelevant provided their respective brackets match.
\end{example}

In order to express properties as illustrated in the above example, we extend rules with variables. 

\Paragraph{String rewriting systems with variables}
Consider an infinite set of variables $\Vars$. 
A \emph{string rewriting system with variables (\SRSV)} $\rewV$ is a finite set of rules $\lhs  \to \rhs$ over $\vAlph \cup \Vars$ satisfying
(1)~every variable $X \in \Vars$ occurs at most once in $\lhs$ and  at most once in $\rhs$, and  
(2)~every variable $X \in \Vars$ that occurs in $\rhs$, also occurs in $\lhs$.
We consider \SRSVs as finite representations of infinite SRSs. 
That is, we say that a rule $\lhs \to \rhs \in \rewV$ generates an infinite 
SRS $\sem{\lhs \to \rhs}$ 
consisting of all ground instances obtained by substituting well-matched words for variables, with the same word substituted for each occurrence of a given variable across both $\lhs$ and $\rhs$.
For example, a rule $c X r \to X$ generates an infinite set $\{ c r \to \epsilon, c c r r \to cr, \ldots \}$.
Finally, an \SRSV $\rewV$ generates an SRS $\sem{\rewV}$ defined as the union of $\sem{\lhs \to \rhs}$  over all rules $\lhs \to \rhs$ from $\rewV$.

\begin{remark}
String rewriting rules can be considered as terms under the unary signature (all symbols have arity at most one) with a single variable.
However, the variables in rules that we consider do not correspond to standard variables in the term-rewriting setting, 
but to context variables.   
\end{remark}

\Paragraph{Properties of SRSs with variables}
The notions defined for SRSs carry over to SRSs with variables by applying the definition to the generated SRS.
That is, we say that a rule with variables $\lhs \to \rhs$ is
\begin{itemize}
    \item consistent (resp., positively/negatively consistent) with a language $\lang$ if and only if 
 the generated SRS $\sem{\lhs \to \rhs}$ is consistent (resp., positively/negatively consistent) with $\lang$,
 \item matching-preserving (resp., well-matched) if and only if 
 the generated SRS $\sem{\lhs \to \rhs}$ is matching-preserving (resp., well-matched).
\end{itemize}
The listed properties extend to SRSs with variables, by requiring that every rule is consistent, well-matched, e.t.c.    
To check whether a rule with variables is matching-preserving (resp., well-matched) it suffices to delete variables, i.e.,
a rule with variables $\lhs \to \rhs$  is matching-preserving (resp. is well-matched)
it holds if and only if
the rule without variables obtained by deleting all variables from $\lhs \to \rhs$ is matching-preserving (resp. is well-matched).


\Paragraph{Well-matched instantiations}
We restrict variables to range over well-matched words only, since non-well-matched instantiations need not be matching-preserving.
Consider the rule $c X r \to X$ that is even well-matched and hence
every rule in $\sem{c X r \to X}$ is well-matched.
However, 
substituting $X$ with $r c$ yields $c r c r \to r c$, which is not even matching-preserving, since $c r c r$ is well-matched, while $r c$ is not.
Furthermore, as discussed in the following section,  restricting to well-matched instantiations facilitates deciding consistency with a given \DVPA.
