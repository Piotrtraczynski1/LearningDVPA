# Review 3


We agree with most detailed remarks, but we oppose the following points:

- Extending results to non-deterministic automata

Our results do not extend to non-deterministic VPA. 
There is a simple reduction of the universality problem, which is EXPTIME-complete for VPA, 
 to checking consistency for rules without variables. In line 314, we explain that positive consistency
 amounts to checking emptiness of L(A_1) \ L(A_2), which is EXPTIME-complete, for A_1, A_2 being non-deterministic VPA.

- Lemma 8. Order of variables

You are correct that the reason for coNP-hardness is the different order of variables.
We attempted to construct a polynomial-time algorithm for rules with a common variable order, but were unsuccessful.
We suspect it to be coNP-hard as well.


- "489: The rule does not make much sense. In the described setting, already w_i -> v_i is a valid (and shorter) rule for every i."

This rule uses k variables, whereas w_i -> v_i has no variables. 
We studied the problem of minimal consistent rules with variables. 
Note that it is "harder" for a rule with variables to be consistent
since all its well-matched instances have to be consistent.
Initially, we expected the lower bound to be exponential, before showing a polynomial upper bound.
