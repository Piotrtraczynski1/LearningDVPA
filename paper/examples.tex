\subsection{Implementation}

\Paragraph{Baseline}
As a baseline for our experiments, we have implemented a polynomial-time active learning algorithm for \DVPA~\cite{mfcs22}.
The algorithm from~\cite{mfcs22} assumes a stronger teacher model than the typical MAT; it allows queries extended membership queries
that read as well modify the stack content and hence it depends on a concrete \DVPA rather than \VPL.
On the positive side, the learning algorithm works in polynomial time.

\subsection{Experiments}

We have conducted a series of experiments to address the following research questions:
\begin{enumerate}[Q1]
\item What is the average reduction in the number of equivalence queries avoided using advice? 
Are there any features that influence performance of the advice mechanism?
\item Does the use of the advice mechanism lead to better running time of the whole learning setup 
(i.e., the runtime of the learning algorithm plus the runtime of the probabilistic equivalence oracle)? 
\item Does the use of the advice mechanism lead to higher reliability of the learning setup in presence of
the probabilistic equivalence oracle?
\end{enumerate}

\Paragraph{Benchmarks}
We have generated random benchmark \DVPA with multiple parameters such as the size of the alphabet 
(the number of call, local and return letters are specified separately), the size of the stack alphabet, 
number of states, density of transitions (the ratio of defined transitions to all possible transitions, where
undefined ones are treated as valid transitions to a special sink state \todo{Is 'density of transitions' worth mentioning?})
and the ratio of accepting states among all states. 
We have observe that the number of states has the main infulence on the number of equivalence queries, while
the size of the alphabet or the size of the stack alphabet have negligable impact on the number of required equivalence queries.
We have also implemented random generation of a \DVPA that is (a)~a Single-Entry VPA (SEVPA),  (b)~Multi-Entry VPA (MeVPA), 
(c)~Call-Driven Automata (CDA), and Expanded Call-Driven Automata (eCDA). 
Moreover, we have implemented random generation  of \DVPA that are consistent which each type of advice discussed above:
\begin{itemize}
    \item For \emph{well-matched commutativity}, we generate two \DVPA over disjoint alphabets, take the product of these automata, where
undefined transitions are defined as self-loops. 
    \item For \emph{well-matched cancellation}, we generate a \DVPA $\aut$ randomly over the alphabet without letter $c,r$. 
    Then, $\aut$ is extended to $\aut'$ over the alphabet including $c,r$ such that the rule $c X r \to \epsilon$ is consistent.   
    \item For \emph{Well-matched idempotency}, we generate a SEVPA with the condition that returning to 
    the same module does not change the state, which implies that $c c X rr \to c  X r$ is consistent.
\end{itemize}

\Paragraph{Exact equivalence check}
In this section, we present the reduction in the number of equivalence queries achieved by using SRS,
as well as its impact on the average execution time required to process a single DVPA in our setup.

The exact equivalence check dominates the overall computation time. When no inconsistency is detected during the SRS phase,
the equivalence check must be performed twice. At the same time, the SRS phase is computationally cheaper, as it operates
on smaller automata and requires constructing products of reduced size, which may lead to performance improvements.

The experimental setup assumes an alphabet consisting of three symbols of each type. For the commutativity property,
both automata use alphabets of this structure. The number of stack symbols is set to five for cancellation and idempotency,
and to sixteen for commutativity. The results are aggregated over $100$ randomly generated automata for each number of states.
The number of well-matched words used for SRS was limited to 40.

\Paragraph{Probabilistic equivalence check}
In this section, we evaluate two aspects: the impact of SRS on reliability and the reduction in processing time.
Since SRS enables performing equivalence testing in two stages, we compare it against a baseline without
SRS using two configurations: $10^5$ and $2 \cdot 10^5$ random words.
The second configuration is intended to verify whether the improvements observed with
SRS are not merely a consequence of a larger number of random tests.

The experimental setup consists of $10^4$ randomly generated automata, each with $10$ to $25$ states.
The input alphabet was generated individually for each automaton, with slight variations depending on the considered property.
In particular, the number of symbols of each type ranged from $2$ to $5$, while the number of stack symbols ranged from $2$ to $16$.
The number of well-matched words used for SRS was limited to $200$.

\Paragraph{Summary}
The presented results show a reduction of approximately $20\%$ in the number of queries for commutativity and idempotency,
and around $50\%$ for cancellation rules. In our setup, the time measurements indicate that the use of SRS
generally increases the overall execution time. However, this trend changes with the size of the automata,
for the largest instances, SRS begins to provide performance improvements.
This suggests that its usefulness increases with the size of the automaton.

Based on experiments for DFA~\cite{ecai25}, we expected the most significant gains for commutativity.
However, the obtained results differ substantially from these expectations. We hypothesize that the method
used to generate the automata did not allow the full potential of this rule to be observed.

The reliability results do not show a significant improvement.
Nevertheless, SRS improves the success ratio of the heuristic approach, while introducing only a negligible time overhead.
This suggests that SRS can be considered a useful feature when a heuristic approach is employed.
