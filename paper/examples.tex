\subsection{Implementation}

\Paragraph{Baseline}
As a baseline for our experiments, we have implemented a polynomial-time active learning algorithm for \DVPA~\cite{mfcs22}.
The algorithm from~\cite{mfcs22} assumes a stronger teacher model than the typical MAT; it allows queries extended membership queries
that read as well modify the stack content and hence it depends on a concrete \DVPA rather than \VPL.
On the positive side, the learning algorithm works in polynomial time. 

\Paragraph{The advice mechanism}
We have extended the baseline algorithm with the advice mechanism, which intercepts equivalence queries before
passing them to the the equivalence oracle performs the consistency check. 
The advice is given via rewrite rules with a {single} variable, and to make it more efficient instead of 
the exact algorithm presented in the paper, we substitute for the variable one of few selected well-matched words.

\subsection{Experiments}

We have conducted a series of experiments to address the following research questions:
\begin{enumerate}[Q1]
\item What is the average reduction in the number of equivalence queries avoided using advice? 
Are there any features that influence performance of the advice mechanism?
\item Does the use of the advice mechanism lead to better running time of the whole learning setup 
(i.e., the runtime of the learning algorithm plus the runtime of the probabilistic equivalence oracle)? 
\item Does the use of the advice mechanism lead to higher reliability of the learning setup in presence of
the probabilistic equivalence oracle?
\end{enumerate}

\Paragraph{Benchmarks}
We have generated random benchmark \DVPA with multiple parameters such as the size of the alphabet 
(the number of call, local and return letters are specified separately), the size of the stack alphabet, 
number of states, density of transitions (the ratio of defined transitions to all possible transitions, where
undefined ones are treated as valid transitions to a special sink state \todo{Is 'density of transitions' worth mentioning?})
and the ratio of accepting states among all states. 
We have observe that the number of states has the main infulence on the number of equivalence queries, while
the size of the alphabet or the size of the stack alphabet have negligable impact on the number of required equivalence queries.
We have also implemented random generation of a \DVPA that is (a)~a Single-Entry VPA (SEVPA),  (b)~Multi-Entry VPA (MeVPA), 
(c)~Call-Driven Automata (CDA), and Expanded Call-Driven Automata (eCDA). 
Moreover, we have implemented random generation  of \DVPA that are consistent which each type of advice discussed above:
\begin{itemize}
    \item For \emph{well-matched commutativity}, we generate two \DVPA over disjoint alphabets, take the product of these automata, where
undefined transitions are defined as self-loops. 
    \item For \emph{well-matched cancellation}, we generate a \DVPA $\aut$ randomly over the alphabet without letter $c,r$. 
    Then, $\aut$ is extended to $\aut'$ over the alphabet including $c,r$ such that the rule $c X r \to \epsilon$ is consistent.   
    \item For \emph{Well-matched idempotency}, we generate a SEVPA with the condition that returning to 
    the same module does not change the state, which implies that $c c X rr \to c  X r$ is consistent.
\end{itemize}
